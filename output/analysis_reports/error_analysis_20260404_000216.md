### ❌ 失败用例 1: 单进程栈内存使用率测试

**错误摘要**: Failed to download smaps for PID 99999

### 🔍 根本原因
**：传入的进程 ID 变量 `${TEST_PID}` 的值为 `99999`。在 Linux 系统中，`/proc` 目录下不存在 PID 为 99999 的进程（该 PID 通常表示无效或已终止的进程），导致 SSH 客户端在执行 `download_proc_files` 时无法读取 `/proc/99999/smaps` 文件，从而抛出异常。

### 🔧 修复方案
**方案一：修正测试数据（推荐，快速修复）**
检查 Robot Framework 测试用例中 `${TEST_PID}` 变量的赋值逻辑。确保传入的是目标测试进程的真实 PID，而不是占位符或错误的数值。

**方案二：增强代码健壮性（推荐，长期改进）**
在 `采集并分析进程栈内存` 方法中，增加对 PID 存在性的校验，避免无效 PID 导致的异常。

**代码修改片段（StackMonitorKeywords.py 第 52 行附近）：**

```python

    if not self.ssh_client:

    # 【新增】增加 PID 存在性校验

        raise Exception(f"进程 ID {pid_int} 不存在或已终止，无法下载 smaps")

    # 下载原始数据
    local_files = self.ssh_client.download_proc_files(pid_int)
```
