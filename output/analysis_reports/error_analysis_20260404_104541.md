### ❌ 失败用例 1: 单进程栈内存使用率测试

**错误摘要**: Failed to download smaps for PID 99999

### 🔍 根本原因
**：
    *   **进程不存在**：错误信息 `Failed to download smaps for PID 99999` 表明，目标进程 ID 为 99999。在 Linux 系统中，`/proc/[pid]/smaps` 文件仅存在于该进程运行期间。如果 PID 99999 不存在（例如它是测试数据中的占位符、拼写错误，或者进程已退出），`download_proc_files` 方法将无法读取文件，导致异常抛出。
    *   **权限不足**：虽然 SSH 用户通常有权限读取 `/proc`，但极少数情况下可能因用户组限制导致无法读取特定进程的内存映射。

### 🔧 修复方案
**步骤 1：修正测试数据变量**
首先检查 Robot Framework 测试脚本中 `${TEST_PID}` 变量的来源。它应该是一个动态获取的进程 ID（例如通过 `ps` 命令查找特定进程名），而不是一个固定的、可能不存在的数字（如 99999）。

**步骤 2：增强代码健壮性**
修改 `StackMonitorKeywords.py`，在下载文件前增加进程存在性检查和异常捕获。

**代码修改建议**：

```python
# 修改 StackMonitorKeywords.py 中的 采集并分析进程栈内存 方法

def 采集并分析进程栈内存(self, 进程ID):
    """
    采集指定PID的栈内存数据并进行分析
    :param 进程ID: 目标进程ID
    :return: (栈上限KB, 实际使用KB)
    """
    if not self.ssh_client:
        raise Exception("请先建立SSH连接")

    pid_int = int(进程ID)
    
    # [新增] 1. 检查进程是否存在
    try:
        check_cmd = f"ps -p {pid_int} -o pid= > /dev/null 2>&1"
        self.ssh_client.exec_command(check_cmd)
        # 注意：这里假设 exec_command 返回码为0表示成功，实际需根据 SSH 客户端实现调整
        # 或者直接尝试读取文件，利用 Python 的 FileNotFoundError
    except Exception as e:
        raise Exception(f"进程 PID {pid_int} 不存在或无法访问: {str(e)}")

    # [修改] 2. 增加异常捕获，提供更清晰的错误信息
    try:
        # 下载原始数据
        local_files = self.ssh_client.download_proc_files(pid_int)
        logger.info(f"栈内存原始数据已下载：smaps={local_files['smaps']}, limits={local_files['limits']}")
    except FileNotFoundError:
        raise Exception(f"无法读取进程 PID {pid_int} 的 /proc/{pid_int}/smaps 文件，请检查进程是否存在或权限是否足够。")
    except Exception as e:
        raise Exception(f"下载进程 {pid_int} 的内存数据失败: {str(e)}")

    # 分析数据
    analyzer = StackAnalyzer(
        smaps_file_path=local_files['smaps'],
        limits_file_path=local_files['limits']
    )
    limit_kb, usage_kb = analyzer.analyze_smaps()
    self.analysis_result = (pid_int, limit_kb, usage_kb)
    
    return limit_kb, usage_kb
```
