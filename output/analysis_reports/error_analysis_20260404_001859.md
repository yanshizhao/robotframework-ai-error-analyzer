### ❌ 失败用例 1: 单进程栈内存使用率测试

**错误摘要**: Failed to download smaps for PID 99999

### 🔍 根本原因
**：在尝试通过 SSH 下载目标进程（PID 99999）的 `/proc/[pid]/smaps` 文件时失败。
*   **具体分析**：错误信息 `Failed to download smaps for PID 99999` 表明 `self.ssh_client.download_proc_files(int(进程ID))` 方法执行异常。这通常由以下原因导致：
    1.  **进程不存在**：PID 99999 可能是一个占位符，或者该进程在测试执行时已经崩溃或终止。
    2.  **权限不足**：SSH 用户没有读取 `/proc` 下特定文件系统的权限。
    3.  **文件传输错误**：SSH 连接虽然建立，但在文件传输过程中出现网络中断或文件路径错误。

### 🔧 修复方案
**建议方案：增强异常捕获与 PID 验证**

在 `采集并分析进程栈内存` 方法中添加异常处理逻辑，并增加对 PID 是否存在的初步检查。

**代码修改片段：**

```python
def 采集并分析进程栈内存(self, 进程ID):
    """
    采集指定PID的栈内存数据并进行分析
    :param 进程ID: 目标进程ID
    :return: (栈上限KB, 实际使用KB)
    """
    if not self.ssh_client:
        raise Exception("请先建立SSH连接")

    # --- 修复开始 ---
    try:
        # 1. 尝试下载原始数据
        local_files = self.ssh_client.download_proc_files(int(进程ID))
        logger.info(f"栈内存原始数据已下载：smaps={local_files['smaps']}, limits={local_files['limits']}")
    except FileNotFoundError as e:
        logger.error(f"PID {进程ID} 不存在或无法访问: {str(e)}")
        raise Exception(f"无法下载 smaps for PID {进程ID}，请确认进程是否存在且权限足够")
    except Exception as e:
        logger.error(f"下载文件时发生未知错误: {str(e)}")
        raise
    # --- 修复结束 ---

    # 分析数据
    analyzer = StackAnalyzer(
        smaps_file_path=local_files['smaps'],
        limits_file_path=local_files['limits']
    )
    limit_kb, usage_kb = analyzer.analyze_smaps()
    self.analysis_result = (int(进程ID), limit_kb, usage_kb)
    return limit_kb, usage_kb
```
