### ❌ 失败用例 1: 单进程栈内存使用率测试

**错误摘要**: Failed to download smaps for PID 99999

### 🔍 根本原因
**：
    目标进程 PID `99999` 在目标服务器上**不存在**或**无权访问**。
    代码尝试通过 SSH 下载 `/proc/99999/smaps` 文件，但该目录不存在，导致 `download_proc_files` 方法抛出异常。

### 🔧 修复方案
**方案：在下载文件前增加进程存在性校验**

在 `采集并分析进程栈内存` 方法中，调用 `download_proc_files` 之前，先通过 SSH 检查目标 PID 是否存在。

**代码修改建议：**

```python
    def 采集并分析进程栈内存(self, 进程ID):
        """
        采集指定PID的栈内存数据并进行分析
        :param 进程ID: 目标进程ID
        :return: (栈上限KB, 实际使用KB)
        """
        if not self.ssh_client:
            raise Exception("请先建立SSH连接")

        # --- 修复代码开始 ---
        # 1. 检查进程是否存在
        check_cmd = f"test -d /proc/{进程ID}"
        check_result = self.ssh_client.run_command(check_cmd)
        
        if check_result.return_code != 0:
            raise Exception(f"进程 {进程ID} 不存在或无权访问。请检查PID是否正确。")
        
        logger.info(f"进程 {进程ID} 存在，开始下载数据...")
        # --- 修复代码结束 ---

        # 下载原始数据
        local_files = self.ssh_client.download_proc_files(int(进程ID))
        logger.info(f"栈内存原始数据已下载：smaps={local_files['smaps']}, limits={local_files['limits']}")

        # 分析数据
        analyzer = StackAnalyzer(
            smaps_file_path=local_files['smaps'],
            limits_file_path=local_files['limits']
        )
        limit_kb, usage_kb = analyzer.analyze_smaps()
        self.analysis_result = (int(进程ID), limit_kb, usage_kb)
        ...
```
